所有 JS 和后续知识点的内容扩充，必须按 Phase 1 的标准执行：

1. **以大厂真题驱动** — 先搜 2026 年字节/阿里/腾讯真实面试题，确认考点是真正的高频
2. **内容深度** — 每个知识点达到原理层 + 手写层 + 对比层 + 面试题层
3. **页面结构** — Hero 标题 → TOC 导航 → Section Cards（概念讲解 + 可运行 Demo） → 动手实验区（可编辑代码） → 面试问答（⭐分级）
4. **可运行代码** — 每个核心概念配一个可运行的代码 Demo
5. **面试导向** — 面试 Q&A 覆盖高频问题，提供回答框架和技巧

**Why:** 用户目标是 4 年经验冲刺 15k+ 一二线前端岗，纯概念讲解不够

**How to apply:** 每次新增 JS/浏览器/工程化等知识点页面时，先搜大厂真题，再按上述标准构建内容
