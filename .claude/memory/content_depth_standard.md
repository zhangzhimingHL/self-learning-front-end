所有知识点页面必须达到 4 年前端 / 一二线 15k+ 面试深度，不能停留在"是什么"层面。

## 深度标准
- **原理层**：不能只讲用法，要讲底层机制（如作用域 → 执行上下文/词法环境/[[Environment]]）
- **手写层**：高频手写题必须有实现 + 变化题
- **对比层**：相似概念必须做对比表（如防抖 vs 节流、var/let/const）
- **源码层**：涉及框架的概念要关联源码（如 Vue 响应式、虚拟 DOM diff）
- **面试回答框架**：每道面试题提供"面试官想听到的回答结构"

## 反面例子
- 只讲"闭包是什么" 不讲 [[Environment]] → ❌ 不够深
- 只讲 var/let/const 区别 不讲执行上下文创建阶段 → ❌ 不够深

## 正面例子
- 闭包 → 形成条件 + [[Environment]] + 词法环境 + 引用持有 → GC 分析 → 内存注意
- this → 默认/隐式/显式/new 绑定 + 优先级 + 箭头函数(词法 this) + 严格模式差异

**Why:** 用户目标是 4 年经验一二线 15k+，前端市场卷，必须做到原理层深度才能竞争

**How to apply:** 每次新建知识点页面/补充内容时，先问自己"这够 15k+ 深度吗？" 原理层 + 手写层 + 对比层 + 源码层四层覆盖至少两层
